### HIGH (product blockers / core)
- **Verified bootstrap gate (initialization as comply-or-fail)** [HIGH / build — next build]. Refined June 10 from a corpus-read fence into a VERIFIED bootstrap checklist: the seat does not come online (may not propose or act) until every orientation step PASSES a deterministic check — not merely "the seat was instructed to orient." The open ritual is an instruction the seat can skip (and did, June 10: a seat that skipped the full read behaved as if perpetually orienting, asserted capabilities absent / limits present without checking, cost the operator a morning). A verified gate removes the choice. Each step returns a checkable result: (1) MANUAL — read OPERATING_MANUAL.md, return a known fact from it (e.g. the current courier allowlist count); (2) QUEUE — read agent_queue head, state the single current next action; (3) CORPUS — query live table counts + activity arc, surface a state summary a deterministic check ratifies (mirrors the migration-drill acceptance test); (4) HANDS — hit the courier (e.g. /op/mailbox_peek) and get the expected response, proving the seat actually has the hands it thinks it has; (5) ENGINE STATE — both engines' running/idle status. ANY check fails -> seat reports "NOT ORIENTED" and refuses to act, instead of proceeding on stale state. This is the Tetraform's comply-or-fail logic turned INWARD on the seat's own init — don't trust the self-report, verify it. PREREQUISITE for trustworthy fractal workers: you cannot safely fan out to N worker seats unless each PROVES it oriented before touching anything — the gate is what makes the workers you're about to multiply trustworthy. The gate and the fractal-worker speedup are the same project. Pairs with the worker manual + open-ritual discipline.
